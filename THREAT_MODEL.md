# Модель Угроз (Threat Model) Athlete Pro

## 1. Контекст Архитектуры

Athlete Pro — это **Offline-first** PWA приложение, использующее:

- **Frontend**: Vanilla JS (ESM), Service Workers, IndexedDB.
- **Backend**: Node.js (Express) с ограниченным API (в основном для связи с AI и интеграциями).
- **Storage**: IndexedDB (клиент), Supabase / Firebase (для опциональной облачной синхронизации).
- **Framework & Methodology**: **Solid 9** (Surgical Fixes) + **#brain** (Квартет ИИ-субагентов для аудита).

## 2. Поверхность Атаки (Attack Surface)

1. **DOM и Пользовательский Ввод**: инъекции через вредоносные названия тренировок или ответы от AI.
2. **Локальное Хранилище (IndexedDB)**: кража PII (метрик, веса) при физическом доступе к разблокированному устройству.
3. **Рассинхронизация Данных (Data Loss / Race Conditions)**: потеря данных при жестком закрытии вкладки браузера, переполнение кэша или сброс системного времени (LWW Skew).
4. **API Endpoints**: DDoS-атаки или исчерпание лимитов (Billing Exhaustion) на эндпоинтах взаимодействия с Claude/Gemini.
5. **Service Worker**: Web Cache Poisoning и Cache DOS (переполнение памяти мусорными ссылками).
6. **Цепочка Поставок (Supply Chain)**: уязвимости в сторонних NPM пакетах.

## 3. Идентифицированные Угрозы и Меры Противодействия

| **Data Exposure (Data at Rest)** | Кража метрик из IndexedDB                             | Physical Access, Browser Exploits | AES-256-GCM шифрование PII-данных (Metrics) через фоновый Web Worker перед записью в IndexedDB.                                                                                                  |

Блокирует код только при наличии High/Critical уязвимостей в `production` зависимостях.                                                          |

## 4. Остаточные Риски (Residual Risks)

- **Компрометация AI Провайдера**: если API Claude/Gemini будет взломано и начнет присылать вредоносные Markdown блоки, система полагается исключительно на нашу локальную санацию (`esc()`). Необходимо постоянно поддерживать актуальность санации.
- **Physical Device Takeover**: Полный захват разблокированного телефона. AES-ключ хранится сессионно или в устройстве, поэтому полный дамп памяти может его скомпрометировать (принятый риск для PWA).

## Phase 4: CI/CD Мониторинг & Audit Logs

- **SAST (Static Application Security Testing):** Интеграция \eslint-plugin-security\ для анализа AST (Абстрактного синтаксического дерева). Запускается автоматически через \pre-push\ хук, блокируя коммиты с потенциальным Prototype Pollution, XSS или небезопасным использованием RegExp.
- **Audit Logs:** Минималистичный логгер с C++-ускоренной обфускацией PII. Замена \logger.js\ с глубокого обхода (O(N) по глубине) на передачу кастомного replacer в \JSON.stringify\. Это предотвращает блокировку Event Loop и утечку \password\, \weight\, \cookie\ в логи.
