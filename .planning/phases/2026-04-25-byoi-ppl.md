## Decisions (locked)

### Training program structure
3 blocks per day:
- **Block I** — Heavy compounds (chest/back/legs) — same in Week A/B
- **Block II** — Secondary work — same in Week A/B
- **Block III** — Arms (Push/Pull only) — **swaps between Week A/B**
- **Core** — checklist-only, visual formality, **no DB persistence**, **no tonnage**

### Week A vs Week B (Block III only)

| Day  | Week A (default)               | Week B (swap)                  |
|------|--------------------------------|--------------------------------|
| Push | Biceps (alt/hammer/preacher)   | Triceps (rope/OH ext/**Close-Grip Bench or Dips**) |
| Pull | Triceps (rope/OH ext) + Shrugs | Biceps (alt/hammer/preacher) + **Shrugs stay**     |
| Legs | Shoulders (lateral/upright/rear-pec) | unchanged                |

### Data model (locked)
```
localStorage:
  ap-custom-plan-A     // Week A plan (replaces existing ap-custom-plan)
  ap-custom-plan-B     // Week B plan
  ap-week-mode         // 'A' | 'B' — current active week
  ap-core-checklist    // [{ name, day, order }] — flexible Core items
```
Migration: existing `ap-custom-plan` → `ap-custom-plan-A` on first load. Both A/B populated by PPL preset.

### Core block defaults
- **Push/Pull:** Hanging Leg Raises + Hyperextensions
- **Legs:** Dead Bug + Plank
- Visual checkboxes only. No weight/reps/tonnage. Drag&drop reorder, add/remove.

### isUnilateral rule
Marked exercises (e.g. Iso-Lateral Seated Row) get `tonnage × 2` in `_updateLiveStats` and `completeSession`. Already implemented.
