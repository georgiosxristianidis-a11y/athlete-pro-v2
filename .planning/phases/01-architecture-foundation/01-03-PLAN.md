# Plan 03: Claude Store/View Split + JSDoc

## Objective
Split `js/claude.js` (758 lines) into `js/claude.store.js` (data, state, heatmap computation, SSE fetch) and `js/claude.view.js` (FAB, panel DOM, SVG rendering, chat UI). Add JSDoc annotations to all public APIs. After this plan, the original `js/claude.js` file is deleted and the two new files are imported by `js/app.js`.

## Tasks

```js
/**
```

The `_fetchCoach` function currently reads `_context` and `_chatHistory` from the Claude IIFE closure. In the store, these are now accessed via the module-level variables directly. The function should:

- Set `_streaming = true`
- Make the SSE fetch to `/api/coach` with the context and history
- Call `onText(chunk)` for each SSE data event
- Push `{ role: 'assistant', content: fullResponse }` to `_chatHistory` on completion
- Call `onDone()` on completion
- Set `_streaming = false` in finally block

Important: This file must NOT contain any DOM references (`document.*`, `innerHTML`, `addEventListener`, `getElementById`).
