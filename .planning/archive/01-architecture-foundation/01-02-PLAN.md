# Plan 02: ES Module Entry Point + JSDoc on db.js

## Objective
Create the ES Module infrastructure by extracting Nav/Toast into `js/shell.js`, creating the `js/app.js` entry point with explicit window bridge assignments, converting `js/db.js` to an ES Module with full JSDoc annotations, and switching `index.html` to load `<script type="module" src="js/app.js">`. After this plan, the app runs identically to before but all JS files are loaded as ES Modules through a single entry point, and `db.js` has complete type documentation.

## Tasks

### Task 02-02-01: Create js/shell.js — Extract Nav and Toast

<read_first>
- index.html — lines 821-865 contain the Nav IIFE (assigned to `window.Nav`) and Toast IIFE (assigned to `window.Toast`) inside the DOMContentLoaded handler
</read_first>

<action>
Create the file `js/shell.js` as an ES Module exporting `Nav` and `Toast`.

1. Extract the Nav IIFE body from index.html lines 821-841. Convert it from an IIFE to a plain object with the same API. The `handlers` map references `Dashboard`, `Workout`, `Analytics`, `Profile`, and `renderBodyStats` — these will be globals set by `app.js`, so they are accessible via `window.*` at call time. Do NOT import them in shell.js (that would create circular deps). Instead, reference them via `window.Dashboard`, `window.Workout`, etc. in the handlers map.

The Nav object must have:
- `go(id)` function — identical logic to current: toggle `.active` classes, scroll to top, call handler
- `current()` function — returns current screen id

2. Extract the Toast IIFE body from index.html lines 844-865. Convert it from an IIFE to a plain object.

The Toast object must have:
- `show(msg, type = 'info', duration = 2800)` function — identical logic to current

3. Export both:
```js
export const Nav = { go, current };
export const Toast = { show };
```

Important: `Toast` accesses `document.getElementById('toast-wrap')`. This element must exist when `show()` is called. Since ES Modules are deferred by default and the element is in the HTML body, this is safe. However, the `wrap` variable must be resolved lazily (inside `show()`, not at module evaluation time) to be safe:
```js
function show(msg, type = 'info', duration = 2800) {
  const wrap = document.getElementById('toast-wrap');
  // ... rest of logic
}
```
</action>

### Task 02-02-02: Convert js/db.js to ES Module with JSDoc

<read_first>

</read_first>

<action>
Modify `js/db.js` to become an ES Module:

1. Remove `'use strict';` at the top (ES Modules are strict by default).

2. Add `// @ts-check` as the very first line for VS Code type checking.

3. Add JSDoc `@typedef` declarations near the top of the file (after the constants section):

```js
/**
 * @typedef {{ id: number, type: 'push'|'pull'|'legs', timestamp: number,
 *   duration: number, tonnage: number,
 *   exercises: Array<ExerciseRecord> }} WorkoutRecord
 *
 * @typedef {{ name: string, sets: Array<SetRecord> }} ExerciseRecord

 * @typedef {{ weight: number, reps: number, rpe: number|null, done: boolean }} SetRecord
 *
 * @typedef {{ id: string, value: number, timestamp: number }} OneRMRecord
 *
 * @typedef {{ id: number, weight: number, height: number, bmi: number, timestamp: number }} MetricsRecord
 */
```

4. Add JSDoc `@param` and `@returns` annotations to ALL public functions. Minimum set:
   - `openDB()` — `@returns {Promise<IDBDatabase>}`
   - `Workouts.save(session)` — `@param {WorkoutRecord} session` / `@returns {Promise<number>}`
   - `Workouts.getAll()` — `@returns {Promise<WorkoutRecord[]>}`
   - `Workouts.getLast(n)` — `@param {number} n` / `@returns {Promise<WorkoutRecord[]>}`
   - `Workouts.deleteById(id)` — `@param {number} id` / `@returns {Promise<void>}`
   - `Workouts.getLastByType(type)` — `@param {'push'|'pull'|'legs'} type` / `@returns {Promise<WorkoutRecord|undefined>}`
   - `Workouts.weeklyVolume()` — `@returns {Promise<number>}`
   - `Workouts.monthlyVolume()` — `@returns {Promise<number>}`
   - `Workouts.monthlyCount()` — `@returns {Promise<number>}`
   - `Workouts.pplTonnage()` — `@returns {Promise<{push: number, pull: number, legs: number}>}`
   - `Workouts.weeklyTrend(weeks)` — `@param {number} weeks` / `@returns {Promise<Array<{label: string, kg: number}>>}`
   - `OneRM.epley(weight, reps)` — `@param {number} weight` `@param {number} reps` / `@returns {number}`
   - `OneRM.update(name, weight, reps)` — `@param {string} name` `@param {number} weight` `@param {number} reps` / `@returns {Promise<void>}`
   - `OneRM.get(name)` — `@param {string} name` / `@returns {Promise<OneRMRecord|undefined>}`
   - `OneRM.getAll()` — `@returns {Promise<OneRMRecord[]>}`
   - `Metrics.bmi(weight, heightCm)` — `@param {number} weight` `@param {number} heightCm` / `@returns {number}`
   - `Metrics.save(weight, heightCm)` — `@param {number} weight` `@param {number} heightCm` / `@returns {Promise<void>}`
   - `Metrics.latest()` — `@returns {Promise<MetricsRecord|undefined>}`
   - `Metrics.getAll()` — `@returns {Promise<MetricsRecord[]>}`
   - `Settings.set(key, value)` — `@param {string} key` `@param {*} value` / `@returns {Promise<void>}`
   - `Settings.get(key, fallback)` — `@param {string} key` `@param {*} [fallback]` / `@returns {Promise<*>}`
   - `Settings.getAll()` — `@returns {Promise<Object<string, *>>}`
   - `Backup.export()` — `@returns {Promise<string>}`
   - `Backup.import(jsonStr)` — `@param {string} jsonStr` / `@returns {Promise<boolean>}`
   - `clearAll()` — `@returns {Promise<void>}`

5. Remove the auto-init line (line 374: `openDB().catch(...)`). The `app.js` entry point will call `openDB()` explicitly.

6. Replace the last line `const DB = { ... };` with an export:
```js
export const DB = { Workouts, OneRM, Metrics, Settings, Events, Backup, clearAll, openDB };
```

7. Also add a named export for `openDB` so `app.js` can import it directly:
```js
export { openDB };
```
</action>

### Task 02-02-04: Create js/app.js — Entry point with window bridge

<read_first>
- index.html — lines 790-891 contain the bootstrap script with DOMContentLoaded handler, clock, network status, Nav, Toast, DB init, and service worker registration
- js/shell.js — the Nav and Toast exports just created
- js/db.js — the `openDB` export
</read_first>

<action>
Create the file `js/app.js` as the single ES Module entry point:

```js
// @ts-check
/* ════════════════════════════════════════════════════════
   app.js — Athlete Pro entry point (ES Module)
   Imports all modules, bridges globals for onclick, boots the app
   ════════════════════════════════════════════════════════ */

import { DB, openDB } from './db.js';
import { Timer } from './timer.js';
import { Nav, Toast } from './shell.js';
import { Dashboard } from './dashboard.js';
import { Workout, RestTimer } from './workout.js';
import { Analytics } from './analytics.js';
import { Profile } from './profile.js';
import { Claude } from './claude.js';

/* ── Bridge: expose to window for onclick="" handlers ── */
window.DB = DB;
window.Nav = Nav;
window.Toast = Toast;
window.Dashboard = Dashboard;
window.Workout = Workout;
window.RestTimer = RestTimer;
window.Analytics = Analytics;
window.Profile = Profile;
window.Claude = Claude;
```

Then add the bootstrap logic that was previously in the inline `<script>` in index.html. This includes:
- Clock setup (updateClock + setInterval)
- Network status (online/offline listeners)
- openDB().then(...) boot sequence
- Claude.renderFAB() call after DOM ready
- Service worker registration

```js
/* ── Clock ── */
const clockEl = document.getElementById('status-time');
function updateClock() {
  clockEl.textContent = new Date().toLocaleTimeString('en', {
    hour: '2-digit', minute: '2-digit', hour12: false,
  });
}
updateClock();
setInterval(updateClock, 30000);

/* ── Network status ── */
const pill = document.getElementById('status-pill');
const pillText = document.getElementById('status-text');
function setOnline() { pill.className = 'status-pill online'; pillText.textContent = 'Online'; }
function setOffline() { pill.className = 'status-pill offline'; pillText.textContent = 'Offline'; }
window.addEventListener('online', setOnline);
window.addEventListener('offline', setOffline);
if (navigator.onLine) setOnline(); else setOffline();

/* ── Boot ── */
openDB()
  .then(() => {
    if (!Workout.init()) {
      Dashboard.load();
    }
  })
  .catch((err) => {
    console.error('[boot] DB failed', err);
    Toast.show('Storage unavailable', 'error');
  })
  .finally(() => {
    setTimeout(() => {
      document.getElementById('loading')?.classList.add('hidden');
    }, 350);
  });

/* ── Claude FAB ── */
Claude.renderFAB();

/* ── Service Worker ── */
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('sw.js').catch(() => {});
}
```

Also import body-stats, plate-calc, supabase-check, and db-firebase if they have exports that need window bridging:
```js
import { FirebaseDB } from './db-firebase.js';
import './supabase-check.js';
import './body-stats.js';
import './plate-calc.js';
```

For `renderBodyStats`, if it is a global function used in Nav handlers, bridge it:
```js
import { renderBodyStats } from './body-stats.js';
window.renderBodyStats = renderBodyStats;
```

Note: The `DOMContentLoaded` wrapper is NOT needed. ES Modules with `type="module"` are deferred by default — they execute after DOM parsing, equivalent to DOMContentLoaded timing.
</action>
