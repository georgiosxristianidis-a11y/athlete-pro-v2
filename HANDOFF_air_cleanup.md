## Карточка 1 — BUG-NAV: системная кнопка «назад» закрывает PWA — ✅ DONE 2026-07-02 (6637719)

Источник: полевой фидбек (брат Gio). Первый баг от реального пользователя — высший приоритет.
Итог: History API в shell.js (pushState/popstate/replaceState-корень); unit 214/214, e2e 39/39, превью-цепочка home→train→stats→назад→назад OK. Остаток гейта: ручная проверка на телефоне после деплоя. Идея вдогонку: UI-кнопка «назад» (Apple-паттерн, top-left шеврон на сабэкранах) — отдельная микро-карточка, обсудить дизайн.

- **ЦЕЛЬ:** тап «назад» на телефоне возвращает на предыдущий экран, а не выходит из приложения. Реализация: в `Nav.go` (js/shell.js) добавить `history.pushState({screen}, '')` при навигации + слушатель `popstate` → `Nav.go(state.screen)` без повторного pushState. На корневом экране (s-home) «назад» может закрывать приложение — это нормальное поведение.
- **ГДЕ СТОП:** работает цепочка home → экран → назад → home на телефоне (или в preview с эмуляцией). Открытые модалки/шторы «назад» закрывать НЕ обязательно (это отдельная карточка, если захочется).
- **НЕ ТРОГАТЬ:** View Transitions логику в Nav.go (там гард D3 из NEXT_SESSION), sw.js, экраны.
- Гейт: npm test + e2e smoke + ручная проверка на телефоне.
